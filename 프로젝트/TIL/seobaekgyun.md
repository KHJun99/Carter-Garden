## 시행 착오 및 해결 방안

### 1. RFID 인식 방식 선택 고민
- 처음엔 이미지 인식을 고려했으나 구현이 너무 어려움
- 바코드는 구현이 쉽지만 주제에서 벗어남
- RC522 RFID 리더로 결정했으나 인식 거리가 3~5cm로 짧음

**해결 방안**
- 카트 입구를 깔때기 형태로 좁혀서 물건이 리더기 근처를 반드시 통과하도록 하드웨어 설계
- 한 번에 한 개씩 넣도록 UX 설계
- 소프트웨어가 아닌 하드웨어 가이드로 해결

### 2. 물건 빼기 인식 문제
- RC522는 태그가 사라졌을 때 자동 감지가 어려움
- 물건을 뺄 때 RFID를 한 번 더 인식시키는 방식은 UX가 불편함

**해결 방안**
- MVP: 장바구니 화면에서 수동으로 항목 제거
- 추가 옵션: 무게 센서 + UI 확인 조합
- UHF 리더기로 업그레이드하면 자동화 가능하지만 예산 고려해 RC522로 진행

### 3. 역할 분담 모호함
- 처음엔 개인별로 모호하게 나눴음
- 의존성 관계를 고려 안 함
- Jetson에 모든 기능을 넣으면 메모리 부족 우려

**해결 방안**
- Critical Path 기반 우선순위 설정

### 4. 디스플레이 연결 방식
- 유선 HDMI 연결은 이동성 제약
- 사용자가 카트 옆에서 터치하면 카트가 움직여야 하는데 케이블 문제

**해결 방안**
- WiFi 기반 무선 연결로 확정
- Flask + WebSocket으로 실시간 상태 업데이트
- 스마트폰 또는 고정형 디스플레이로 제어

## 시행 착오 및 해결 방안

### 1. Git 인증 및 push 실패
- GitLab에서 username/password 인증 실패
- Personal Access Token 발급 후 Password에 토큰 입력으로 해결

### 2. Git 원격 저장소 충돌
- 로컬과 원격 히스토리가 달라서 push 거부됨
- git pull origin master --allow-unrelated-histories로 병합 후 push 성공

### 3. GitLab Protected Branch 정책
- master 브랜치에 force push 시도 시 pre-receive hook declined 에러
- develop 브랜치 생성 후 Merge Request 방식으로 우회
- 또는 git merge develop2 -X theirs로 로컬 병합 후 push

### 4. Git 인증 반복 입력 문제
- push할 때마다 username/password 입력 요구됨
- git config --global credential.helper store 설정으로 해결
- 또는 URL에 토큰 포함: https://0000322:토큰@lab.ssafy.com/...

### 5. Jetson OS 설치 중단 우려
- 설치 중 퇴근 시간이 되어 전원 끄기 고민
- 설치 중 전원 차단 시 파일 시스템 손상 위험 학습
- 전원 유지한 채 다음날 확인하기로 결정

### 6. Jetson 메모리 부족 우려
- Jetson에 모든 기능을 넣으면 메모리 부족 가능성

- 웹 서버(키오스크), RFID 등은 라즈베리파이로 분산

**해결 방안**
- Jetson: 자율주행, SLAM, AI 처리 (지연성 최소화 필요)

## 시행 착오 및 해결 방안


### 1. 마트 레이아웃 복잡도 과다
- 처음엔 실제 마트와 유사하게 복잡하게 설계하려 함
- SLAM 맵핑 난이도 상승 및 Navigation 정확도 저하 우려


**해결 방안**
- 3~4개 상품 코너 + 계산대 + 주차 구역 정도로 최소화
- 테스트 후 점진적으로 복잡도 증가


### 2. 휠 및 모터 선택 기준 불명확
- 다양한 제품군 존재하지만 선택 기준 모호
- 토크, RPM, 전압, 가격 등 고려 요소 많음


**해결 방안**
- 최대 적재 하중 실험으로 필요 토크 역산
- 예산 범위 내에서 최대한 안전한 토크 선택


### 3. 메카넘 휠 vs 일반 휠 선택
- 메카넘 휠은 전방향 이동 가능하지만 비용 높고 제어 복잡
- 일반 휠은 저렴하고 안정적이지만 회전 반경 큼


**해결 방안**
- 축소 마트 환경에서는 회전 반경 충분히 확보 가능


### 4. Jetson 메모리 부족 우려
- Jetson에 모든 기능을 넣으면 메모리 부족 가능성
- 자율주행과 같은 메모리 집약적 작업은 Jetson 전담
- 웹 서버(키오스크), RFID 등은 라즈베리파이로 분산


**해결 방안**
- Jetson: 자율주행, SLAM, AI 처리 (지연성 최소화 필요)
- 라즈베리파이: 웹 서버, RFID, 센서 처리
- 병렬 처리 가능하도록 보드 분리
