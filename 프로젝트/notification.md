# Git 커밋 규칙

## 예시
- [feat]0113-강지영: 프로젝트 초기 세팅 및 폴더 구조
- [docs]0114_배유진: README 작성 및 설치 가이드 업데이트
- [fix]0114_배유진: 상품 중복 추가 오류 수정
- [test]0115_강지영: 결제 기능 테스트 코드 작성

## Type 종류
- feat: 새로운 기능 추가
- fix: 버그 수정
- docs: 문서 수정
- test: 테스트 코드 추가/수정

## 기본 규칙
1. 날짜는 MMDD (월일) 형식
2. 시간은 HHMM (24시간) 형식
3. Type을 맨 앞에 [대괄호]로 감싸기
4. 이름 뒤에 콜론(:) 후 작업 내용 간단히
5. 50자 이내로 작성

## 푸시 전 체크
- 코드 테스트 완료
- 불필요한 파일 제외
- Pull 먼저 받았는지 확인
